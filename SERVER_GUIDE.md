# 服务器选择指南

## 📋 概述

部署应用通常需要一台服务器，但有很多选择，包括免费和付费选项。

## 🆓 免费选项

### 1. 本地电脑作为服务器

**优点：**
- 完全免费
- 适合测试和学习
- 完全控制

**缺点：**
- 需要保持电脑开机
- 需要配置公网 IP 或内网穿透
- 不适合生产环境
- 电费和网络稳定性问题

**适用场景：** 个人测试、开发学习

---

### 2. 免费云服务（有限制）

#### Vercel / Netlify（仅前端）
- **免费额度：** 前端静态网站免费
- **限制：** 不支持后端 Node.js 服务
- **适合：** 仅部署前端

#### Railway（推荐）
- **免费额度：** $5/月免费额度
- **特点：** 支持 Docker，易于部署
- **链接：** https://railway.app

#### Render
- **免费额度：** 有限制的免费套餐
- **特点：** 支持 Node.js，自动部署
- **链接：** https://render.com

#### Fly.io
- **免费额度：** 3 个共享 CPU 实例
- **特点：** 全球边缘部署
- **链接：** https://fly.io

#### Heroku（已停止免费套餐）
- **现状：** 不再提供免费套餐
- **价格：** 从 $5/月起

## ⚠️ 注意事项

### 1. 备案问题（国内服务器）
- 如果使用国内服务器，需要域名备案
- 备案通常需要 7-20 天
- 可以使用 IP 地址访问（无需备案）

### 2. 网络问题
- 国内服务器：国内访问快，国外可能慢
- 国外服务器：国外访问快，国内可能慢
- 建议：根据主要用户群体选择

### 3. 安全配置
- 修改默认 SSH 端口
- 使用密钥登录
- 配置防火墙
- 定期更新系统

### 4. 数据备份
- 定期备份数据库
- 使用云服务商的快照功能
- 配置自动备份
